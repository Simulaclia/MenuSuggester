IDEについて

IDEに関しては自由とします。
ただし推奨はEclipseで、それ以外はノンサポートとします。

変数名について

基本的には自由とします。
ただしある程度何のための変数なのかわかる英語にしてください。

コードの記述について

ループ用の変数など明らかにわかる物以外の変数や処理などに関しては基本的にコメントアウトでどのような目的の処理なのか簡潔に記述してください。
また、コードを記述する際に参考にした記事がある場合はわかりやすい場所にURLを記述しておいてください。
詳しくはコード記述例を参照してください。

GitHubについて

今回開発にGitHubを利用する予定です。
開発時にはPrivate状態ですが、発表の際にPublic状態にする予定です。
開発するコードやドキュメントに開発者の個人情報を残さないようにお願いします。

お気に入り機能と作成履歴機能について

お気に入り用のCSVを別に用意し、お気に入りをしたときに上から追記することとします。
より良い実装が可能な場合は再度検討します。

インデントについて

コードの体裁はEclipseの自動整形機能によるものとします。
"設定"→"Java"→"エディター"→"保存アクション"から"保存時に選択したアクションを実行"にチェックを入れ、"ソース・コードのフォーマット"にチェックを入れることでコードの保存時に自動で整形されます。

稼働時間について

基本的に開発は授業時間中のみに行うこととします。
授業時間外の開発を禁止するものではありませんが非推奨です。時間外割増賃金なども発生しません。
ただし班都合での時間外開発はこの限りではありません。ただしその場合も時間外割増賃金は発生しません。

班員技術サポートサービスについて

技術的に困った場合は班員サポートサービスを利用することが可能です。
対応は手が空いている人がすることとします。
繁忙期にはおまたせする可能性があります。ご了承ください。
なお、班員技術サポートサービスは授業時間内のみの稼働となっています。

コーディング規約について

基本的に今までの授業で習ったコーディング規約に従うこととします。
