IDEについて

IDEに関しては自由とします。
ただし推奨はEclipseで、それ以外はノンサポートとします。

変数名について

基本的には自由とします。
ただしある程度何のための変数なのかわかる英語にしてください。

コードの記述について

ループ用の変数など明らかにわかる物以外の変数や処理などに関しては基本的にコメントアウトでどのような目的の処理なのか簡潔に記述してください。
また、コードを記述する際に参考にした記事がある場合はわかりやすい場所にURLを記述しておいてください。
詳しくはコード記述例を参照してください。

GitHubについて

今回開発にGitHubを利用する予定です。
開発時にはPrivate状態ですが、発表の際にPublic状態にする予定です。
開発するコードやドキュメントに開発者の個人情報を残さないようにお願いします。

CSVファイル構成について

料理名,料理国名,料理ジャンル,2人前価格,料理所要時間カロリー,タンパク質量,脂質量,炭水化物量,よく使われる食材,主食材1,主食材2,クックパッドの料理名検索リンク
料理国は和食、洋食、中華の三択にします。
料理ジャンルは揚げ物、煮物、焼き物等を記述してください。
価格は10円単位、2人前の価格とします。
料理の所要時間は10分単位とします。
カロリーは1kcal単位とします。
タンパク質、脂質、炭水化物量は0.1g単位とします。それ以下は切り捨てとします。
記述する料理は基本的に夕食として食べる料理とします。
よく使われる食材は料理に利用されているよく使われるメイン食材を入力することとします。
主食材1、主食材2はあまり利用されない食材も含めたメイン食材を入力することとします。

お気に入り機能と作成履歴機能について

お気に入りや作成履歴用のCSVを別に用意し、お気に入りや作成をしたときに上から追記することとします。
より良い実装が可能な場合は再度検討します。

インデントについて

コードの体裁はEclipseの自動整形機能によるものとします。
"設定"→"Java"→"エディター"→"保管アクション"から"保存時に選択したアクションを実行"にチェックを入れ、"ソース・コードのフォーマット"にチェックを入れることでコードの保存時に自動で整形されます。
